# Описание бизнес-логики проекта SJK SmartView

## 1. Ценностное предложение (Value Proposition)
SJK SmartView решает критическую проблему в цикле продажи DOOH-рекламы (Digital Out-of-Home): **длительное время ожидания визуализации**. 
Вместо того чтобы ждать 1–2 дня работы дизайнера, менеджер Shojiki получает фотореалистичный мокап за секунды прямо во время переговоров. Это повышает конверсию («эффект момента») и снижает нагрузку на дизайн-отдел.

---

## 2. Целевые пользователи
*   **Сэйлз-менеджеры SJK (Вьетнам):** Основные пользователи в офисах Хошимина и Ханоя. Работают "в полях" (планшеты) или из офиса (ноутбуки).
*   **Руководители отделов Shojiki:** Мониторинг активности и эффективности предложений.
*   **Клиенты (Локальные и международные бренды):** Конечные потребители визуального контента во Вьетнаме.

---

## 3. Основные сценарии использования (User Journeys)

### Сценарий А: «Из каталога» (Быстрая продажа)
1.  Менеджер выбирает конкретную локацию (например, "Terra Royal LED") из базы.
2.  Загружает креатив клиента (баннер).
3.  Система применяет **предрассчитанную маску** и параметры перспективы.
4.  Через 1-2 секунды менеджер скачивает готовый файл или отправляет ссылку клиенту.

### Сценарий Б: «С улицы» (Новый объект / Expert Mode)
1.  Менеджер фотографирует рекламную конструкцию на телефон.
2.  Загружает фото в приложение + баннер клиента.
3.  **AI-Диспетчер** отправляет мультимодальный запрос (BG + Asset) в Gemini 3.
4.  Система использует методологию **SCHEMA** для нативного поиска экрана и бесшовного вписывания креатива с учетом освещения.
5.  Менеджер получает результат в течение 5–10 секунд.

---

## 4. Алгоритмический стек (Реализовано)

Система использует **гибридную архитектуру** для обеспечения максимальной стабильности и реализма.

### Стратегия генерации:
1.  **Primary Path (Native AI Expert)**: Основной путь для премиальных мокапов. 
    - Прямое взаимодействие с **Gemini 3 Pro Image** через `google-genai`.
    - Используется **SCHEMA Methodology** для управления «мышлением» модели.
    - Нативная гармонизация: учет бликов, теней и текстуры поверхности без OpenCV-масок.
2.  **Fallback Path (Standard)**: Используется как черновик или при сбоях AI.
    - **OpenCV Perspective Warp (Homography)** выполняется локально на сервере.
    - Мгновенный результат, служащий геометрическим драфтом.

---

## 5. Инфраструктурные особенности
*   **Hybrid Storage (Firebase):** Хранение оригиналов и результатов в облаке (регион `asia-southeast1`).
*   **Firebase Auth:** Безопасный доступ для менеджеров Shojiki.
*   **Serverless GPU (Modal):** Тяжелые вычисления (YOLO, OpenCV) запускаются только по требованию, минимизируя затраты (Pay-as-you-go).
*   **CORS Proxy:** Собственный прокси `/api/image-proxy` на стороне Next.js для бесперебойной загрузки текстур.

---

## 6. Метрики успеха (KPI)
1.  **Lead-to-Proposal Time:** Сокращение времени подготовки КП с 2 часов до 2 минут.
2.  **Manager Adoption:** Доступность инструмента на любом смартфоне (PWA).
3.  **Visual Authenticity:** Фотореализм уровня профессионального ретушера за счет учета перспективы и освещения.
